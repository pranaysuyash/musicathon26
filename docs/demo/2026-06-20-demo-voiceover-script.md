## Recording notes

- **Pause 1 sec** between scenes for breath + edit points.
- **Don't rush.** The numbers ("ninety percent", "fifty-seven songs") are the credibility layer. Land them cleanly.
- **Total read time:** 4:30 ± 15 seconds.
- **Edit out:** any "um", "uh", false starts. Keep the takes clean. 30s buffer is enough for 1-2 cuts.
- **Equipment:** A quiet room. AirPods mic is fine for the voiceover. If you have a USB condenser, use it.

## What's NOT in the demo

- **No queries.** The audience isn't going to type; we show the curated paths.
- **No code.** This isn't a code review; the jury doesn't need to see Python or TypeScript.
- **No datasets.** The 442-song corpus is the corpus; we don't defend it.
- **No founder story.** The product speaks.

## What IS in the demo

- **9 distinct screens.** Different enough that the viewer sees scope.
- **Real numbers.** 442 songs, 15 events, 67 nodes, 82 edges, 95% lyrics coverage.
- **Real artifacts.** Lyric lines, edge evidence, year signals — not screenshots of Lorem Ipsum.
- **One narrative arc.** Song → signals → events → graph → themes → comparison → regions → health → close.
